Buffer Depth — KV Prefix Fill Over Time

The KV ring holds four live buffers simultaneously — one per stage transition. As workers process items, the counts shift. Stage C is the bottleneck (simulates LLM latency), so the B→C buffer fills while C→D drains slowly.

The bottleneck is visible in the buffers: the B→C queue fills while Stage C workers are saturated. In a real system this is where you'd add workers or reduce LLM call complexity. The signal is in the KV prefix depth — no external metrics pipeline needed.
§3

KV Architecture — The Buffer Is Already Everywhere

In a conventional pipeline, workers talk to an external queue (Redis, Kafka, SQS). The queue is a separate system. In an Agentic Flow Network, the buffer is the KV ring — already replicated to every node as part of the cluster.

KV Key Layout

pipeline/stage-a/{id} ← seeded by coordinator (200 items)
pipeline/stage-b/{id} ← written by Stage A workers (parse result)
pipeline/stage-c/{id} ← written by Stage B workers (enrich result)
pipeline/stage-d/{id} ← written by Stage C workers (score result)
pipeline/done/{id} ← written by Stage D workers → also in Postgres
pipeline/claiming/{id} ← coordinator claim lock (TTL=30s, prevents double-dispatch)

# deploy: redis server, celery worker, flower
# monitor: redis queues + celery inspect

from celery import Celery

app = Celery('pipeline',
    broker='redis://redis:6379/0',
    backend='redis://redis:6379/1')

@app.task
def score_article(payload):
    time.sleep(SLEEP)  # LLM call
    result = compute_score(payload)
    # result goes to Redis backend
    # consumed by next .delay() call
    return result

# Caller:
score_article.delay(payload)
# Dead-letter: Redis separate config
# Backpressure: Celery concurrency arg
# Routing: Celery queues config
Mycelium AFN worker + skill handler 
# requirements: mycelium-py
# deploy: nothing extra
# monitor: KV prefix depth

from mycelium import MyceliumAgent

agent = MyceliumAgent("127.0.0.1", 8300)
agent.advertise_capability("stage_c", "worker")

async for req in agent.rpc_serve("stage_c.score"):
    payload = json.loads(req.payload)
    time.sleep(SLEEP)  # LLM call
    result  = compute_score(payload)
    # write to KV ring — gossips everywhere
    agent.set(f"pipeline/stage-d/{id}", result)
    agent.rpc_respond(req, b"ok")
    # Dead-letter: TTL expiry (automatic)
    # Backpressure: in_flight tracking
    # Routing: capability resolution
