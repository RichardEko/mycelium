The New Reality02 / 25
AI agents are not microservices
The infrastructure playbook for microservices breaks down when your units of compute are ephemeral, capability-driven, and self-organizing.
Microservices World
Static topology
Services have fixed names and addresses. Service discovery finds them. They don't change shape at runtime.
Register → serve → deregister
Explicit lifecycle. The registry must be told when something starts and when it stops. Crashes leave stale entries.
Coordinator-centric
One system knows the topology. Everyone asks it. It becomes the bottleneck and the failure mode.
AI Agent World
Dynamic capability sets
An agent's role emerges from what it can do — and that changes as models load, tasks complete, and resources shift. Discovery must keep up.
Ephemeral by nature
Agents spawn, exhaust their purpose, and die. The fleet must heal around them automatically. No operator involvement.
Peer-to-peer intelligence
Agents need to find each other, negotiate capability, and coordinate — without a broker in the critical path that becomes a single point of failure.
The infrastructure hasn't kept up. The tools we have were built for stateless HTTP services, not for fleets of reasoning agents with dynamic capability surfaces.

Theoretical Foundations05 / 25
Not retro-fitted to theory.
Built from it.
Mycelium began with John Holland's Complex Adaptive Systems framework as its design blueprint — then asked what that looks like running in Rust at production scale.
Holland — Signals & Boundaries (MIT Press, 2012)
Signals
Information that travels between agents and triggers responses. Selective — only agents that recognise the signal act on it.
Boundaries
Constraints that define which agents interact. Boundaries create structure from the inside — no external authority assigns them.
Tags & Aggregation
Agents carry tags (declared properties). Matching tags cause spontaneous aggregation into higher-order structures — emergent groups with no coordinator.
Emergence over prescription
Macro-level behaviour arises from micro-level rules. The designer specifies interaction primitives, not outcomes.
Mycelium — The Implementation
Layer II — Signal Mesh
Scoped ephemeral events with admission boundaries. Only nodes that advertise the matching capability receive group-scoped signals. Holland's signals, made concrete.
Capability Groups & Opacity
Group boundaries emerge from capability declarations. Opacity composition enforces admission without a gatekeeper. The boundary is distributed, not policed.
Capability Advertisement (Tags)
advertise_capability("compute","gpu") is a tag. resolve() matches tags. Emergent groups self-assemble from nodes sharing compatible tag sets.
No coordinator, by design
The cluster topology is never prescribed. It emerges from gossip, TTL expiry, and capability matching — exactly as Holland's CAS model predicts it should.
Also standing on: Grassé (1959) Stigmergy — agents coordinate through environment modification, not direct communication. The gossip KV layer is the environment. · Demers et al. (1987) Epidemic Algorithms — the gossip substrate. · Kulkarni et al. (2014) Hybrid Logical Clocks — causal ordering without NTP.

Self-Organization07 / 25
The fleet knows
what it can do
Capabilities advertised, not registered
Each node writes its capabilities as TTL'd KV entries that gossip through the cluster. Groups form automatically from nodes sharing compatible capabilities. No administrator defines the groups.
Dead nodes vanish automatically
When a node stops refreshing its TTL entries, they expire. The cluster heals in under one health interval. No deregister call. No stale entries. No operator alert.
Late joiners catch up fully
Anti-entropy sync on reconnect means a restarted node recovers the entire cluster state from its peers within seconds. No replay log. No snapshot restore.
The cluster knows what it can do — and it always has an up-to-date view, on every node, without asking a coordinator.
Agentic Flow Networks08 / 25
The cluster is the queue
Conventional pipelines pin workers to static stages. AFN workers shift to wherever demand exists.
Opacity Backpressure — automatic
Busy workers mark themselves opaque. The capability resolver skips them. Overloaded stages draw in idle workers from elsewhere — zero config.
KV ring IS the pipeline buffer
Stage items are KV entries. Any worker that matches the capability can pick them up. No queue server, no topic config, no dead-letter queue.
Push → Pull — the last decision-maker, removed
The KV-ring buffer still lets a coordinator push work to a predicted worker. The mycelium-tuple-space companion flips this: workers take when ready, so no component models anyone's load. Linda's pull pattern, on a coordinator-free substrate. (Appendix A10)
Beyond the demo
The demo uses identical workers that each carry all stage logic — vivid, but it undersells the model. In production: specialist workers advertising only one stage, heterogeneous teams shipping independent capability sets, zero-downtime rollout of score_v2 alongside score_v1. The mesh assembles the topology from whatever is running. No operator involvement.
Demo assumption ≠ Mycelium's requirement. Fluid allocation works across a fully heterogeneous fleet — different images, different teams, different capabilities. Topology emerges bottom-up.

Three-layer Architecture
Layer I — KV Store
Last-write-wins gossip over TCP. HLC causal ordering. TTL per key. Anti-entropy on reconnect. Bloom-filtered dedup. WAL persistence.
Layer II — Signal Mesh
Ephemeral scoped events. Broadcast / group / individual scopes. Opacity composition for admission control. Pheromone attenuation at group boundaries.
Layer III — Consensus
Epidemic ballot-voting (CASPaxos-adjacent). No Raft leader. consistent_set, distributed_lock, elect_leader. Cross-group proposals via GroupQuorum.
Single KV substrate. All three layers write to dedicated key prefixes via the same gossip mechanism. cap/ · sig/ · consensus/ · sys/ · mailbox/ · prompts/ — one substrate, many namespaces.
Structural Inversion: AP substrate always available. CP overlay opt-in. Pay for ordering guarantees only where your application demands them.

Consensus — Ballot-Serialized Agreement
Epidemic ballot voting — no leader
CASPaxos-adjacent. Proposals gossip through the cluster. No Raft leader. Concurrent writes to the same key are totally ordered by ballot number.
Cross-group federation
cross_group_propose + GroupQuorum — multiple groups each act as an independent voting bloc. Models multi-AZ durability gates and hierarchical approval.
Why not Raft: Raft requires a stable leader. Agent fleets have constant churn. Epidemic consensus tolerates node turnover without stalling progress.
// Ballot-serialized KV write agent.consensus().consistent_set("key", val).await?; let v = agent.consensus().consistent_get("key"); // Distributed lock with fencing token let lock = agent.consensus() .distributed_lock("job-42", ttl_secs=30).await?; println!("token: {}", lock.token); // monotonic fencing token // Leader election let leader = agent.consensus().elect_leader("shard-0").await?; // Cross-group proposal — both blocs must reach quorum agent.consensus().cross_group_propose("config", val, vec![GroupQuorum { group: "llm", quorum: 0.5, veto: false }, GroupQuorum { group: "compliance", quorum: 0.5, veto: true }]).await?;

Security — Transport · Identity · Integrity
mTLS transport --features tls
Mutual TLS on every gossip TCP connection. External attackers cannot inject or intercept frames. Zero overhead when the feature flag is off — all TLS code compiles away.
Ed25519 node identity
Each node generates an Ed25519 keypair at startup. Same keypair serves as mTLS identity, gossips into sys/identity/{node}, and signs consensus payloads.
Wire v11 — KV write signing
WireMessage::SignedData — Ed25519 signature over key, value, timestamp, tombstone flag, and sender. A compromised node cannot inject false KV entries.
// Enable mTLS — one config line GossipConfig { tls: Some(TlsConfig::default()), // or: gateway_auth_token = Some("token") .. }
Threat model: mTLS stops external injection. Wire v11 signing stops a compromised cluster node from injecting modified KV frames. Both layers required for full integrity.
HTTP gateway: no auth by default — bind to loopback or set gateway_auth_token. /health, /ready, /metrics intentionally public for load-balancer probes.

Appendix A924 / 25
Semantic Coordination — Contracts in the Mesh
Capability schema versioning
Each capability carries an optional schema_id gossip-propagated with the entry. CapFilter::with_schema("acme-ml/v2") matches only that version — prevents silent mismatches between teams.
Payload schemas in the ring
with_input_schema / with_output_schema embed JSON Schema strings in the gossip-propagated capability entry. Callers inspect the contract from resolve() results — no separate lookup.
Sender authorization
signal_rx_from(kind, trusted) delivers only signals from trusted senders — enforced at the fan-out layer. Closes the semantic-injection vector for LLM agents processing signal payloads as prompts.
// Advertise with schema + embedded contracts let cap = Capability::new("llm", "chat") .with_schema_id("llm-chat/v1") .with_input_schema(r#"{"type":"object","required":["prompt"]}"#) .with_output_schema(r#"{"type":"object","required":["reply"]}"#); // Resolve with schema filter let providers = agent.capabilities().resolve( &CapFilter::new("llm", "chat").with_schema("llm-chat/v1")); // Trusted-sender subscription let mut rx = agent.mesh() .signal_rx_from("task.assign", vec![orchestrator_id]); // Schema registry agent.schemas().publish_schema("llm-chat/v1", json_bytes).await?; agent.schemas().seed_schemas_from_dir("./schemas").await?;
Fully backward-compatible. All fields use #[serde(default)] — existing capabilities decode without change; old nodes ignore richer entries.
Appendix A1025 / 25
TupleSpace — Pull, Without the Coordinator
A companion crate (mycelium-tuple-space) built entirely on the public API — the constructive proof that the most coordinator-shaped workload, work distribution, needs no decision-maker.
Pull, not push
Producers put items into named stages; workers block on take and receive work exactly when ready. Readiness is announced by the claim itself — there is no model of anyone's load to go stale, so the misrouting that staleness causes in push systems has no variable to inhabit.
The ring IS the failure detector
The serving node is discovered by capability advertisement; its failure is detected by the same TTL evaporation that fades any pheromone; a mirroring standby promotes itself through the same emergent election as every other role. Kill the primary mid-flight and survivors are served — acknowledged items never resurrect.
Durable by construction
Single-lock hot path; a write-ahead log whose Complete record is one indivisible entry, so a stage transition can never half-replay. Compaction bumps a WAL epoch so a secondary's byte-offset replay cursor can never silently dangle.
// Producer — no knowledge of any worker let id = space.put("stage.fetch", payload).await?; // Worker — blocks until work is ready, then claims it let (id, item) = space.take("stage.fetch").await?; // ...do the work... space.complete(id, "stage.parse", next).await?; // atomic ack + advance
Why it matters. Gelernter's tuple space (Linda, 1985) was always the right pull primitive; the classical objection — the shared space is itself a coordinator needing consistency machinery — dissolves when the space is built on a coordinator-free substrate. 1,000 items · 10 producers · 10 workers · exactly-once · zero shared load view.
Composability proof. The crate's only dependency on the core is the public API — pull composes from the existing primitives. Nothing was added to the substrate to make it possible.
