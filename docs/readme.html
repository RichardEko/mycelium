Layer I — Gossip KV Transport

Layer 1

TCP epidemic propagation with last-write-wins conflict resolution. Every key has a TTL. Anti-entropy sync delivers missed updates to late joiners automatically.

use mycelium::{GossipAgent, GossipConfig, NodeId};
use std::sync::Arc;
rust
let mut config = GossipConfig::default();
config.bootstrap_peers = vec!["127.0.0.1:7947".parse()?];

let agent = Arc::new(GossipAgent::new(NodeId::new("127.0.0.1", 7946)?, config));
agent.start().await?;

// Write — local store always updated; returns false if gossip channel full
agent.kv().set("key", Bytes::from("value"));

// Read
if let Some(bytes) = agent.kv().get("key") { /* ... */ }

// Delete (propagates a tombstone)
agent.kv().delete("key");

// Scan by prefix — capability discovery, pheromone trail reads
let entries = agent.kv().scan_prefix("load/");

// Subscribe — fires on every local or gossiped change
let mut rx = agent.kv().subscribe("load/my-node");
rx.changed().await?;

agent.shutdown().await;

Observability

let stats = agent.system_stats();rust

// dropped_frames > 0 → gossip writes silently lost; raise writer_channel_depth
assert_eq!(stats.dropped_frames, 0);

println!("peers: {}",  stats.peers);          // live peers in ping table
println!("store: {}",  stats.store_entries);   // live KV entries
assert!(stats.gc_alive);                      // tombstone + subscription cleanup
assert!(stats.health_monitor_alive);          // peer pings and eviction
⚠️
Dropped frames diagnostic: dropped_frames > 0 means gossip writes were silently lost. Size writer_channel_depth to N_agents × fan_out (default fan_out = 4). Non-zero shard_queue_depths means gossip workers are falling behind — increase gossip_shards.

Durability-Confirmed Writes — set_with_min_acks

set_async returns after the local write; gossip propagates to peers asynchronously. set_with_min_acks waits until min_acks distinct peers have applied the key, giving a durable acknowledgement before returning. 

// Returns the number of peers that confirmed the write.rust
// Err(QuorumError::Timeout { acks_received }) if < min_acks acked before timeout.
let n = agent.kv().set_with_min_acks(
    "cfg/cluster-epoch",
    Bytes::from("42"),
    2,                           // min_acks from distinct peers
    Duration::from_secs(5),     // timeout
).await?;
println!("confirmed by {} peers", n);

The HTTP gateway exposes the same operation:

POST /gateway/kv/quorumjson
{ "key": "cfg/cluster-epoch", "value_b64": "NDI=", "min_acks": 2, "timeout_secs": 5 }

// Success:  { "ok": true,  "acks_received": 2 }
// Timeout:  { "ok": false, "error": "timeout", "acks_received": 0 }

Layer II — Signal / Boundary Mesh

Layer 2

Signals are ephemeral events that propagate epidemically to every node. Each node holds a local boundary — group memberships — that decides whether it acts on an incoming signal. Forwarding is always unconditional; the boundary only controls local delivery.

use mycelium::{signal_kind, SignalScope};rust

// Group membership
agent.mesh().join_group("nlp");
agent.mesh().leave_group("nlp");

// Emit
agent.mesh().emit("invoke", SignalScope::Group("nlp"), payload);      // non-blocking
agent.mesh().emit_async("invoke", SignalScope::Group("nlp"), payload).await;

// Receive
let mut rx = agent.mesh().signal_rx(signal_kind::INVOKE);
tokio::spawn(async move {
    while let Some(sig) = rx.recv().await {
        // sig.sender, sig.payload, sig.scope, sig.nonce
    }
});

// One-shot request/response
let reply = agent.mesh().signal_once("invoke.result", Duration::from_secs(5), |s| {
    s.nonce == request_nonce
}).await; // Option<Signal>; None on timeout

Opacity vs Inhibition

Opacity — passive, automatic

Opacity is not a feature you call. When handler channels fill under load, admit_prob = 1.0 - fill_ratio. At 60% fill, 60% of System/Group signals are shed. The node still forwards every signal — it simply stops reacting to new arrivals. Individual scope always bypasses opacity.

manage_opacity and OpacityHint add a governor task that tells peers when this node is becoming saturated — they do not change the shedding itself.

Inhibition — active, deterministic

suppress(kind, duration) is called deliberately by your code. For the duration, no signals of that kind are delivered — 100% blocked, not probabilistic. Use for refractory periods and rate limiting.

// Refractory period — block next invocation for 500ms after handling onerust
while let Some(sig) = invoke_rx.recv().await {
    agent.mesh().suppress(signal_kind::INVOKE, Duration::from_millis(500));
    handle_invocation(sig).await;
}

// Rate limit — prevent sync storms
agent.mesh().suppress(signal_kind::DATA_SYNC, Duration::from_secs(5));

// Lift early
agent.mesh().unsuppress(signal_kind::INVOKE);

Layer III — Consensus

Layer 3

Lightweight epidemic two-phase agreement built directly on the signal mesh — no extra wire format, no separate consensus port. Committed values are written to consensus/committed/{slot} and anti-entropy-synced to late joiners automatically.

Protocol: Propose → (votes from group) → Commit → KV committed/{slot}

use mycelium::{ConsensusConfig, ConsensusResult};rust

let _listener = agent.start_consensus_listener();

let cfg = ConsensusConfig { quorum_size: 0, ..ConsensusConfig::default() };
match agent.group_propose("workers", "coordinator", Bytes::from("node-7"), cfg).await {
    ConsensusResult::Committed { slot, value, ballot } => { /* ... */ }
    ConsensusResult::Timeout { .. }    => { /* no quorum */ }
    ConsensusResult::Superseded { .. } => { /* another node won */ }
}

// System-wide proposal
agent.system_propose("global/epoch", Bytes::from("42"), ConsensusConfig::default()).await;

// Subscribe to a slot
let mut rx = agent.consensus_rx("coordinator");
ℹ️
quorum_size = 0 uses floor(N/2) + 1 (simple majority). With the tls feature all consensus payloads are Ed25519-signed; forged ballots are dropped. Without TLS, consensus assumes a trusted domain.

Capability Subsystem

First-class capability advertisement, discovery, demand pressure, and locality-aware routing — all built on the Layer I KV store. No external registry; everything lives under cap/, req/, and gcap/ namespaces and anti-entropy-syncs to late joiners automatically.

Advertising and Resolving

use mycelium::{Capability, CapFilter, CapabilityHandle};rust

// Advertise — periodically reasserts cap/{node_id}/{ns}/{name} in KV.
// Drop the handle to stop; tombstone propagates automatically.
let handle: CapabilityHandle = agent.advertise_capability(
    Capability::new("compute", "gpu"),
    Duration::from_secs(30),
);

// Resolve — snapshot of all matching nodes
let matches: Vec<(NodeId, Capability)> = agent.resolve(&CapFilter::new("compute", "gpu"));

// Watch — debounced push-based; fires when the matching set changes
let mut rx = agent.watch_capabilities(CapFilter::new("compute", "gpu"));
rx.changed().await?;

Requirements

// Declare need — writes req/{node_id}/{ns}/{name} to KVrust
let _handle = agent.declare_requirement(
    CapFilter::new("compute", "gpu"),
    Duration::from_secs(30),
);

let mut rx = agent.watch_requirement(CapFilter::new("compute", "gpu"));
rx.changed().await?;
println!("satisfied: {}", rx.borrow().is_satisfied());

Demand Pressure

demand_pressure = demanding_nodes.len() / max(providers.len(), 1). Pressure > 1.0 means demand outstrips supply. The library never auto-responds — this is a signal for orchestrators and autoscalers.

let status = agent.demand(&CapFilter::new("compute", "gpu"));rust
println!("pressure: {:.2}", status.demand_pressure);

let mut rx = agent.watch_demand(CapFilter::new("compute", "gpu"));
rx.changed().await?;

Emergent Capability Groups

Nodes that share a capability automatically form a named group. The library projects their collective capability under gcap/{group}/… and handles group-level requirement wiring.

agent.define_capability_group(rust
    "gpu-pool",
    CapabilityGroupDef {
        filter:   CapFilter::new("compute", "gpu"),
        provides: vec![Capability::new("compute", "gpu")],
        requires: vec![],
    },
    Duration::from_secs(60),
);

Locality-Aware Resolution

Each node declares a locality_path (["az1", "rack2", "host3"] coarse→fine). resolve_with_locality sorts providers by shared-prefix depth — closest first.

config.locality_path = vec!["az1".into(), "rack2".into(), "host3".into()];rust

// Returns (NodeId, Capability, depth) sorted depth desc
let candidates = agent.resolve_with_locality(
    &CapFilter::new("render", "job"),
    LocalityPreference::PreferShared(0),  // prefer closest; fall back to any
);

// Route to locality-closest provider via wiring
agent.signal_wired_via_locality(
    &CapFilter::new("render", "job"),
    LocalityPreference::PreferShared(0),
    "render-job", payload,
).await;

Service Layer

RPC, bulk transfer, scatter-gather, and durable mailboxes — the service primitives used by language bridges and the MCP integration.

Point-to-Point RPC

// Callerrust
let reply = agent.rpc_call(target, "echo", payload, Duration::from_secs(5)).await?;

// Responder
let mut rx = agent.rpc_rx("echo");
while let Some(req) = rx.recv().await {
    agent.rpc_respond(&req, req.payload());
}

Bulk Payload Transfer

For payloads too large to gossip through every node, bulk_call stages data at a local HTTP endpoint and sends only a lightweight ticket over the mesh.

// Requires http_port set in GossipConfigrust
let reply = agent.bulk_call(target, "process", large_bytes, Duration::from_secs(30)).await?;

Scatter-Gather

Fan out an identical request to multiple targets concurrently; return as soon as min_ok replies arrive.

let results = agent.scatter_gather(targets, "vote", payload, Duration::from_secs(5), 2).await?;rust

Actor/Event Mailboxes

KV-backed durable event delivery. Events survive crashes and are delivered in HLC-causal order.

// Sender (any node)rust
agent.deliver_event(&target_id, "task.result", result_bytes);

// Receiver — at-least-once within TTL; tombstoned after delivery
let (handle, mut rx) = agent.open_mailbox("task.result", 64);
while let Some(event) = rx.recv().await {
    process(&event.payload);
}

Consistency & Ordering Overlay

Overlay

Higher-level coordination primitives — consistent KV, distributed locks, leader election, a replicated append-only log, and reliable signal delivery — all layered on top of the Gossip KV substrate and exposed through the HTTP gateway.

ℹ️
All overlay operations are available via the HTTP gateway at /gateway/… and are wrapped by both the Python and TypeScript SDKs. No extra ports or services required.

Consistent KV & Distributed Locks

consistent_set / consistent_get use a consensus round to guarantee linearisable reads and writes. acquire_lock returns a guard whose token must be passed with every protected write — the lock is released automatically when the guard is dropped or its TTL expires. 

// HTTP — consistent linearisable writejson
POST /gateway/consistent/set
{ "key": "cfg/leader", "value_b64": "bm9kZS0x", "timeout_secs": 5 }

// HTTP — distributed lock (returns guard_id + token)
POST /gateway/lock/acquire
{ "lock_name": "deploy-lock", "ttl_secs": 30 }

DELETE /gateway/lock/release/<guard_id>
# Pythonpython
agent.consistent_set("cfg/leader", b"node-1")
val = agent.consistent_get("cfg/leader")

with agent.acquire_lock("deploy-lock", ttl_secs=30) as lock:
    agent.consistent_set("deploy/state", b"running")

Distributed Log

An HLC-ordered append-only log per namespace. append assigns a monotonic HLC sequence number. scan_log reads a range. subscribe_log streams new entries as an SSE feed. 

POST /gateway/log/append         { "ns": "audit", "value_b64": "…" }json
GET  /gateway/log/scan/audit?from=0&to=1000
GET  /gateway/log/subscribe/audit     # SSE stream
POST /gateway/log/compact         { "ns": "audit", "before_hlc": 1000 }
# Pythonpython
hlc = agent.append_log("audit", b"event data")
entries = agent.kv().scan_log("audit", from_hlc=0, to_hlc=1000)
for entry in agent.kv().subscribe_log("audit"):
    print(entry.hlc, entry.value)

Leader Election

Consensus-backed single-leader election per slot. Returns the winning node-id and the ballot round.

POST /gateway/elect  { "slot": "cluster-leader", "candidate": "<node_id>", "timeout_secs": 5 }json
# Pythonpython
leader = agent.elect_leader("cluster-leader", candidate=agent.node_id)

Reliable Signal Delivery

emit_reliable wraps a signal in a KV-backed envelope with an ACK counter. Recipients confirm delivery; the envelope is cleaned up once all expected ACKs arrive. 

POST /gateway/signal/reliable  { "kind": "deploy", "payload_b64": "…", "scope": "system" }json
# Pythonpython
agent.emit_reliable("deploy", b"payload", scope="system")

Cluster Sharding

shard_for maps a shard key to a deterministic owner node using a consistent hash ring over advertised capability providers. emit_sharded combines the lookup with a targeted signal dispatch — equivalent to Akka Cluster Sharding without a coordinator. 

// Rustrust
// Find the deterministic owner for a shard key
let owner = agent.shard_for("user-12345", &CapFilter::new("session", "store"));

// Emit to the shard owner in one call
let owner = agent.emit_sharded("write", "user-12345", &CapFilter::new("session", "store"), payload).await?;
# HTTP gatewaysh
curl "http://localhost:8300/gateway/shard/session/store?key=user-12345"
# → {"owner":"10.0.0.3:8300"}

curl -X POST http://localhost:8300/gateway/shard/emit \
  -d '{"kind":"write","ns":"session","name":"store","shard_key":"user-12345","payload_b64":"…"}'
# Pythonpython
owner = agent.shard_for("session", "store", "user-12345")
owner = await agent.emit_sharded("write", "session", "store", "user-12345", b"payload")
// TypeScripttypescript
const owner = await agent.shardFor("session", "store", "user-12345");
const owner = await agent.emitSharded("write", "session", "store", "user-12345", payload);
ℹ️
The hash ring is rebuilt on every shard_for call from the live cap/ KV prefix — no coordinator, no rebalance protocol. All nodes seeing the same provider set resolve the same owner deterministically.

Prompt Skills — LLM-backed capabilities via the KV substrate

Optional · llm feature

The llm cargo feature lets any GossipAgent host LLM-backed skills. Prompt templates are stored in the gossip KV store (prompts/{ns}/{name}), replicated cluster-wide, and served by whichever node has a matching backend registered. Callers discover and invoke the skill through the ordinary capability ring — indistinguishable from any other Mycelium skill.

The model field is not in PromptTemplate. Model availability is node-local knowledge; it is baked into the LlmBackend at construction. The LlmResult.model_used field reports what was actually used. 

# Cargo.tomltoml
mycelium = { version = "…", features = ["llm"] }

Registering a skill

// Rust — register on this node; template written to cluster KV (TTL = 1 week)rust
let backend = OpenAiBackend::new(
    "http://localhost:11434/v1", "", "llama3.2",
);
let template = PromptTemplate {
    system: "You are a helpful assistant.".into(),
    user_template: "{{input}}".into(),
    max_tokens: 512,
    temperature: 0.7,
    metadata: Default::default(),
};
let _handle = agent.register_prompt_skill("llm", "chat", template, backend).await?;
// Drop _handle to retract the capability; template persists in KV until deleted.

Calling from Rust

rustlet output = agent.call_prompt_skill(
    "llm", "chat", "Summarise this article…",
    Default::default(), Duration::from_secs(30),
).await?;

HTTP gateway

sh# List all templates visible to this node
curl http://localhost:8300/gateway/prompts

# Write / update a template (propagates to all nodes via gossip KV)
curl -X PUT http://localhost:8300/gateway/prompts/llm/chat \
     -H 'Content-Type: application/json' \
     -d '{"system":"You are a helpful assistant.","user_template":"{{input}}","max_tokens":512,"temperature":0.7}'

# Invoke (blocking)
curl -X POST http://localhost:8300/gateway/llm/call \
     -H 'Content-Type: application/json' \
     -d '{"ns":"llm","name":"chat","input":"What is 2+2?"}'

# Invoke (SSE stream)
curl -N -X POST http://localhost:8300/gateway/llm/stream \
     -H 'Content-Type: application/json' \
     -d '{"ns":"llm","name":"chat","input":"Hello!"}'

Security Model

Mycelium supports mutual TLS (mTLS) on the gossip TCP port via the optional tls cargo feature. When enabled, every peer connection requires a valid cluster-CA-signed Ed25519 certificate.

# Cargo.tomltoml
mycelium = { version = "…", features = ["tls"] }
config.tls = Some(TlsConfig::default()); // auto-generates CA + certs in ./mycelium-tls/rust
What tls providesDetail
mTLS transporttokio-rustls mTLS on every gossip TCP connection; plain nodes cannot connect
Ed25519 node identityTLS cert key = identity keypair; 32-byte verifying key gossiped to sys/identity/{node}
Signed consensus payloadsAll Propose, Vote, Nack, Commit messages are Ed25519-signed; forged ballots dropped
KV write signing (wire v10)WireMessage::SignedData — Ed25519 over hop-invariant fields (excludes TTL); a compromised forwarding node cannot inject or modify KV entries; fail-open for unknown signers (key not yet gossiped)
⚠️
Without the tls feature, Mycelium operates in a trusted domain. A connected peer can send crafted frames to inject KV entries, claim any NodeId, or poison the nonce dedup set. Do not expose gossip ports to untrusted networks without TLS or a network-layer control (firewall, WireGuard, VPC security groups).

scan_prefix uses a prefix index for an O(|segment_keys|) fast path when the prefix segment is known (e.g. "load/", "cap/"). Unknown prefixes fall back to O(store_size). At typical pheromone-trail sizes (100–1 000 entries) the cost is negligible relative to network latency.
