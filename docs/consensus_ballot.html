Guide — what to observe
Layer I Committed values land in KV at consensus/committed/{slot} and gossip-replicate to all peers automatically.
Layer III Two-phase epidemic voting — group_propose broadcasts a ballot, waits for ≥4/7 ACKs, then commits. Competing proposals are superseded; unresolved rounds timeout and retry.
  1. Watch the round-robin proposer change each cycle — each node takes a turn proposing its own value; quorum of 4/7 decides the winner.
  2. A committed row (green) means ≥4 ACKs matched on the same ballot; the value appears in "Committed values" and is KV-synced to all peers.
  3. Click Force propose (node 0) mid-round — node 0 races the scheduled proposer; the slower proposal gets superseded (purple) rather than committed.
  4. A timeout row (amber) means max ballots were exhausted with no quorum — jitter prevents livelock and the next scheduled proposer tries a fresh slot.

group_propose
Epidemic two-phase voting
The proposer broadcasts a PROPOSE ballot over the gossip mesh. Peers that have registered as voters (via start_consensus_listener) reply with their current highest ballot number. When a majority ACK the same ballot, a COMMIT message is broadcast — the value is written to consensus/committed/{slot} and anti-entropy synced to all peers.
let result = agent.group_propose( "cluster", "coordinator", value, cfg, ).await; match result { Committed { slot, value, .. } => {}, Timeout { ballots_tried, .. } => {}, Superseded{ ballot, .. } => {}, }
Committed
Quorum reached — value written
When ⌊N/2⌋+1 nodes ACK the same ballot number, the proposer commits. The committed value is stored in the Layer 1 KV store and epidemic-synced, so even nodes that didn't vote can read it immediately after commit. All 7 nodes act as voters here — quorum requires 4 votes.
// All nodes start listening as voters let _h = agent.start_consensus_listener(cfg); // Read committed value without proposing let v = agent.consensus_get("coordinator"); // Watch for future commits let rx = agent.consensus_rx("coordinator");
Superseded
Another node committed first
If two nodes propose simultaneously, one wins the ballot race and commits. The other receives a Superseded response — a higher ballot number was already committed. It can read the winner's value via consensus_get. Click Force propose during an in-flight round to create contention and observe Superseded appearing in the history.
Superseded { slot, ballot } => { // someone else committed — read their value let winner = agent .consensus_get(&slot) .map(|b| String::from_utf8_lossy(&b) .to_string()); }
