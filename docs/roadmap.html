Engineering Roadmap · 2026

Distributed AI Agent Platform

A swarm of AI agents that signals intent through receptors, discovers capabilities through a shared medium, and moves heavy data over HTTP only when the payload demands it — with no coordinator, no registry, no single point of failure.

Layer 1: Transport — Complete Layer 2: Signal / Boundary Mesh — Complete Layers 3–4: Bulk Transfer + Observability — Planned

Design Philosophy

🧬
Chemical Signalling

Hormones flood the bloodstream. Every cell receives every signal. Specificity comes from receptors — not from routing. This platform works the same way: signals flood the cluster epidemically, boundaries decide who acts.

⚡
Forwarding ≠ Acting

A node outside group::nlp forwards a group-scoped signal at full speed without acting on it. Propagation is guaranteed regardless of whether any intermediate node is in scope. No routing bottlenecks.

🔌
Maximal Decoupling

One transport, one abstraction. Invokers fire a signal; they have no knowledge of how the target is implemented. HTTP only appears in Layer 3, and only for agents handling large payloads.

🐜
Stigmergy

Workers write load state directly into the shared KV medium as pheromone trails. Routing emerges from nodes reading that medium — not from explicit coordination. Stale trails evaporate naturally via embedded timestamps.

Semantics State — what is true right now Events — something must be acted on immediately
Persistence Durable, LWW, anti-entropy synced, visible to late joiners Ephemeral, TTL-bounded, missed if not listening

Layer 2 — Signal / Boundary Mesh

Complete · 94 tests

The Three Scopes

System
admits() → always true

All agents in the cluster act. Global broadcasts, cluster-wide events.

Group
admits() → groups.contains(g)

Only agents whose boundary includes the group name act. NLP agents, data processors, ops monitors — any named cohort. Used for emergent work routing.

Individual
admits() → id == self.node_id

Only the target node acts. Direct replies (invoke.result) are the canonical use. Bypasses opacity — no routing alternative exists.

API — All on GossipAgent, No Wrapper

// Group membership — updates local boundary + gossips grp/<name>/<node_id>
agent.join_group("nlp");
agent.leave_group("nlp");

// Persistent handler — one receiver per kind per task
let mut rx = agent.signal_rx("invoke");
tokio::spawn(async move {
    while let Some(sig) = rx.recv().await {
        let req: InvokeRequest = decode(&sig.payload);
        let result = run_model(&req.payload).await;
        agent.emit(
            "invoke.result",
            SignalScope::Individual(sig.sender),  // reply to specific caller
            encode(InvokeResponse { nonce: req.nonce, result }),
        );
    }
});

// One-shot correlation — register BEFORE emitting so no reply is missed
let nonce = fastrand::u64(..);
let reply_fut = agent.signal_once(
    "invoke.result",
    Duration::from_secs(5),
    move |s| s.nonce == nonce,    // channel registered synchronously on this call
);
agent.emit_async("invoke", SignalScope::Group("nlp".into()), payload).await;
let result = reply_fut.await;   // → Option<Signal>

// Periodic heartbeat — emits signal AND writes pheromone trail each tick
let load_key = format!("load/{}", agent.node_id());
let _handle = agent.advertise(
    "contract.available",
    SignalScope::Group("nlp".into()),
    Duration::from_secs(10),
    move || {
        let state = LoadState { queue_depth: QUEUE.len(), written_at_ms: unix_ms_now() };
        agent2.set(load_key.clone(), encode(&state));  // pheromone trail
        encode(&state)                                    // signal payload
    },
);

Stigmergic Load State — Pheromone Trails

Workers write current load into the Layer 1 KV store at load/<node_id>. Any node reads routing state directly from the shared medium — no local cache, no signal handlers, no coordination. New nodes receive the full picture immediately via anti-entropy sync.

// Read load state for all "nlp" workers at decision time
let candidates: Vec<(NodeId, LoadState)> = agent
    .scan_prefix("load/")
    .into_iter()
    .filter_map(|(k, b)| {
        let state: LoadState = decode(&b)?;
        // Evaporation — discard entries older than 3 × advertise interval
        if unix_ms_now() - state.written_at_ms > 30_000 { return None; }
        let node_id = k.strip_prefix("load/")?.parse().ok()?;
        Some((node_id, state))
    })
    .collect();
Boundary — the Receptor
O(1) on the signal hot path
  • In-memory AHashSet of group names
  • admits(scope) — single hashset lookup, no locks on delivery
  • join_group() / leave_group() update set + gossip KV entry
  • Variable opacity: probabilistic shedding under load
Pheromone Trails
Stigmergy — routing from the medium
  • Workers write load/<node_id> each advertise tick
  • Evaporation via embedded written_at_ms timestamp
  • Crashed workers' trails decay without coordination
  • Late joiners bootstrap immediately via anti-entropy
Emergent Routing
No invoker selects a worker
  • Invoke floods Group scope; workers compete to respond
  • Opaque workers shed signals — stop consuming without coordination
  • First valid reply wins via signal_once
  • On timeout, scan pheromone trails to decide retry vs. escalate
Two-Timescale Load Signalling
Neural + hormonal — both needed
  • Pheromone trail: 5–30 s, durable, visible to late joiners
  • boundary.opaque signal: immediate, connection teardown now
  • Signals handle the gap before the next KV write propagates
  • Use both — they are complementary, not redundant

Phase 3 — Bulk Transfer / AI Execution

3–4 weeks

When payloads exceed practical signal size — multi-KB prompts, model outputs, streaming token responses — a Layer 2 signal carries a ticket. HTTP carries the weight. Agents with only small payloads need no HTTP server at all.

// Layer 2: coordination via signal
layer.signal(
    SignalScope::Individual(target),
    "invoke.bulk",
    encode(BulkInvokeRequest { contract_id, corr_id, input_endpoint }),
);

// Layer 3: target fetches large input, runs model, streams response
// POST /invoke/corr_id  →  {large prompt bytes}
// GET  /result/corr_id  ←  token stream (SSE)

// Layer 2: completion notification back to caller
layer.signal(
    SignalScope::Individual(caller),
    "invoke.result",
    encode(InvokeResponse { corr_id, result_endpoint }),
);
What lives in Layer 3
Only the heavy-payload concerns
  • reqwest HTTP client (rustls-tls, stream feature)
  • Optional HTTP server per agent (axum / hyper)
  • BulkTransferLayer — fetch input, return result
  • SSE / chunked streaming for LLM token streams
  • Distinct Event type for transport-bound events
  • peers() for enumerating reachable nodes by NodeId
What stays in Layer 2
Even for bulk workflows
  • Signal to initiate the request (ticket only)
  • Signal to notify completion
  • Routing via pheromone trails and opacity
  • Group coordination around the invocation

Layer 3 Eventing — Distinct from Layer 2 Signals

Layer 3 introduces a distinct Event type for connection-scoped, ordered, flow-controlled events riding a transport connection. The delivery guarantees differ fundamentally from Layer 2 signals.

PropertyLayer 2 SignalLayer 3 Event
DeliveryEpidemic flood — every nodePoint-to-point transport (SSE, gRPC stream)
OrderingNoneOrdered per connection
ReliabilityBest-effort — can be silently droppedAt-least-once on open stream; failure is explicit
Flow controlProbabilistic opacity sheddingHTTP/2 flow control, TCP backpressure
Use casesWork routing, load coordinationStreaming tokens, cancel signals, progress events

Pattern reuse: the Layer 2 mental model transfers directly. Event kinds reuse the same string constants (invoke.result, boundary.opaque). The "emit to scope, receivers with matching state act" principle applies. Only the delivery substrate changes — the type is distinct to make the guarantee difference explicit in code.

Layers 1 & 2 in Practice

// ── Worker node ───────────────────────────────────────────────────────────
let agent = Arc::new(GossipAgent::new(node_id, config));
agent.start().await?;

// Join group — gossips grp/nlp/<node_id> into the KV store
agent.join_group("nlp");

// Advertise every 10 s — emits signal (fast) + writes pheromone trail (durable)
let load_key = format!("load/{}", agent.node_id());
let _handle = agent.advertise(
    "contract.available",
    SignalScope::Group("nlp".into()),
    Duration::from_secs(10),
    move || {
        let state = LoadState { queue_depth: QUEUE.len(), written_at_ms: unix_ms_now() };
        agent2.set(load_key.clone(), encode(&state));  // pheromone in the medium
        encode(&state)
    },
);

// Handle invocations — emit result back to the specific caller
let mut invoke_rx = agent.signal_rx("invoke");
tokio::spawn(async move {
    while let Some(sig) = invoke_rx.recv().await {
        let req: InvokeRequest = decode(&sig.payload);
        let result = run_model(&req.payload).await;
        agent.emit(
            "invoke.result",
            SignalScope::Individual(sig.sender),
            encode(InvokeResponse { nonce: req.nonce, result }),
        );
    }
});

// ── Invoker node ──────────────────────────────────────────────────────────
// Register BEFORE emitting — channel is live the moment signal_once() returns
let nonce = fastrand::u64(..);
let reply_fut = agent.signal_once(
    "invoke.result",
    Duration::from_secs(5),
    move |s| s.nonce == nonce,
);
// Emit to the group — routing emerges from opacity, no worker selected
agent.emit_async("invoke", SignalScope::Group("nlp".into()), payload).await;

match reply_fut.await {
    Some(sig) => decode(&sig.payload),
    None => {
        // No reply — check pheromone trails to decide retry vs. escalate
        let any_live = agent.scan_prefix("load/").iter()
            .filter_map(|(_, b)| decode::<LoadState>(b))
            .any(|s| unix_ms_now() - s.written_at_ms < 30_000);
        if any_live { retry_with_backoff() } else { Err("no workers") }
    }
}

// ── Upstream node reacting to acute overload ─────────────────────────────
let mut opaque_rx = agent.signal_rx("boundary.opaque");
tokio::spawn(async move {
    while let Some(sig) = opaque_rx.recv().await {
        // Fast-path: tear down connections before the pheromone trail updates
        http_pool.drain_worker(sig.sender);
    }
});

Design Position

Being precise about what is prior art is as important as identifying what is genuinely new. This architecture synthesises established ideas. The differentiation lies in the combination and the context.

Prior Art

ConceptPrior Art
Epidemic gossip propagation Demers et al. 1987 · Cassandra, Consul, Redis Cluster
Scope-filtered pub/sub NATS subjects / queue groups · DDS partitions · MQTT topic trees · SIENA content-based routing (1999)
Application-layer broadcast filtering Implicit in all gossip implementations
Chemical computing as design metaphor Berry & Boudol, Chemical Abstract Machine, 1990
Actor-model group routing Erlang process groups · Akka Cluster distributed pub/sub

What Is Genuinely Differentiated

1 · Broker-less scope filtering
with epidemic delivery guarantees
  • NATS, Kafka require a broker cluster to route by scope
  • Here scope is a pure application-layer filter
  • No routing infrastructure to operate or fail
  • Group signals reach every node; boundary decides action
2 · Group topology as KV state
in the same store, not a separate registry
  • Most systems: separate Consul/etcd + separate message bus
  • Here group membership IS a gossip KV entry
  • Same propagation, same LWW, same visibility
  • Cluster is self-describing — zero extra infrastructure
3 · State and events on one transport
one wire format, two semantic modes
  • KV state: LWW, persistent, queryable, anti-entropy sync'd
  • Signals: TTL-bounded, ephemeral, nonce-dedup'd
  • Typically requires two separate systems
  • One embedded library; no external dependencies
4 · NodeId as contract address
no HTTP endpoint, no service registry
  • LangChain, AutoGPT, CrewAI: invocation routes to HTTP URLs
  • Here the address IS the node's gossip identity
  • No URL to manage, no port to expose
  • The gossip network is the addressing layer

Closest Comparison: NATS.io

NATS is the nearest existing production system with subject-based messaging, queue groups, and cluster-wide gossip membership. The gaps:

DimensionNATS.ioThis design
Infrastructure External NATS server cluster required Embedded library — one Cargo dependency, zero servers
KV state JetStream (separate subsystem) Same gossip store as signals
Capability advertisement Not provided Contract registry built on gossip KV

The specific combination is novel and potentially paper-worthy. Each individual concept — epidemic gossip, boundary filtering, stigmergy, chemical signalling as design metaphor — has prior art. What is genuinely new is the unification of epidemic KV state, ephemeral scoped signals, stigmergic pheromone trails, and emergent load-adaptive routing in a single embedded library, grounded in biological CAS principles as first-class design constraints rather than post-hoc metaphor. The two-timescale coordination model (neural signals + hormonal trails) applied to distributed AI agent orchestration has no direct precedent in the literature.
