Guide
  1. Click any worker to toggle its compute/gpu capability. Workers with the cap glide inside the pool ring; workers without slide back out.
  2. The pool count is computed from gcap/gpu-pool/compute/gpu/{member} entries — written by the single C3 membership task running on each member.
  3. Each blue flash is one signal_wired_via("render-job", …) from a consumer. The resolver picks one or more pool members; the chosen one's tile pulses.
  4. Drain the pool to zero by retracting every worker — consumer outcome flips to Unwired until you add a member back.

define_capability_group
Emergent groups
cap-group/{name} publishes a filter. Every node whose own cap/ advertisements satisfy the filter self-joins via watch_capability_group_definitions and starts writing gcap/{name}/{ns}/{name}/{self}.
let def = CapabilityGroupDef { filter: CapFilter::new("compute", "gpu"), provides: vec![Capability::new("compute", "gpu")], requires: vec![], topology_policy: None, }; let h = agent.define_capability_group( "gpu-pool", def, Duration::from_secs(30), );
signal_wired_via
Resolver picks the provider
One call resolves the wiring AND dispatches the signal. Returns Emitted with the list of providers actually routed to, or Unwired if no provider currently matches the filter.
match agent.signal_wired_via( &filter, "render-job", payload, ) { Emitted { providers } => { .. } Unwired { filter } => { // no provider — queue, retry, alert } }
C3 — task consolidation
One task per group
Each pool member runs a single run_group_membership_task that owns both the gcap/ reassertion ticker AND any group-requirement opacity tracking. Pre-C3 it would have been one task per (provide × require) per group — here, with 10 members each providing one capability and zero requirements, that's a flat 10 tasks instead of 10× anything.
// Each member tombstones every gcap/ // key it wrote when the task exits. async fn run_group_membership_task( .. provides: Vec<Capability>, requires: Vec<CapFilter>, .. );
