quorum_persistent reads sys/quorum/heartbeat/ from Layer 1 KV — evidence that survives supervisor restarts. Kill 3 or more services and watch the circuit open.
quorum_persistent
Restart-safe health evidence
Unlike quorum() which reads in-memory counters reset on restart, quorum_persistent(kind, window) counts distinct senders whose heartbeat evidence is still in the Layer 1 KV store within the window. Kill the supervisor and restart it — the circuit breaker reads immediately from KV, not from a cold in-memory counter.
let window = Duration::from_secs(3); let count = supervisor .quorum_persistent("heartbeat", window); // count = distinct senders in sys/quorum/ // heartbeat/ within the last 3 seconds let open = count < THRESHOLD;
Service heartbeats
System-scope signals
Each service emits a System-scope "heartbeat" signal every second. The supervisor's signal handler records sender identity and timestamp; the Layer 2 quorum track in sys/quorum/heartbeat/{sender} is updated by the signal layer automatically. Kill a service to stop its heartbeat — the evidence expires after 3 seconds and the quorum count drops.
// service emits every second service.emit( "heartbeat", SignalScope::System, Bytes::copy_from_slice( &now_ms().to_le_bytes() ), );
Circuit breaker
Kill services, watch it open
Click Kill on 3 services. The quorum counter drops below 4 within 3 seconds as their evidence expires from the KV window. The circuit opens immediately. Click Restart to bring them back — the first heartbeat each sends updates its KV evidence and the count climbs back up. The circuit closes once quorum is restored.
// supervisor checks every 500 ms let count = sup.quorum_persistent( "heartbeat", window); if count < THRESHOLD { open_circuit(); // reject incoming requests } else { close_circuit(); }
