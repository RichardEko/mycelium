Guide — what to observe
Layer II Opacity Governor — workers self-report channel saturation via pheromone trails in Layer I KV; dispatchers read peer_load and route around saturated nodes.
  1. Watch at rest — dispatchers share load evenly; fast workers (0–3, 150 ms/task) and slow workers (4–7, 700 ms/task) both stay green.
  2. Click Inject burst — slow workers fill their channels immediately; bars turn red and the OPAQUE badge appears.
  3. Within ~300 ms, dispatchers detect opaque workers via peer_load and stop routing to them — fast workers absorb all new tasks.
  4. Slow workers drain passively (no new tasks arrive); bars turn green once fill drops below the 60% threshold, then dispatchers resume routing to them.

manage_opacity
Self-reported load via Layer 2
Each worker calls manage_opacity("task", …), which monitors the worker's "task" signal handler channel fill ratio every 100 ms. When fill exceeds the threshold (60%), the governor emits BOUNDARY_OPAQUE and writes the load state to sys/load/{id}/task — visible to all peers via Layer 1 epidemic sync.
let _h = agent.manage_opacity( "task", SignalScope::System, OpacityHint { threshold: 0.60, hysteresis: 0.20, ..Default::default() }, );
peer_load
Cluster-wide fill map
Dispatchers call peer_load(max_age) every dispatch cycle. It reads all sys/load/ entries (written by workers' governors, synced epidemically), filters by kind="task", and returns fill ratios sorted highest-first. The dispatcher picks the minimum — the least saturated worker available — and routes the next task to it via Individual scope.
let loads = dispatcher.peer_load( Duration::from_secs(5) ); let target = loads.iter() .filter(|(_, k, _)| k == "task") .min_by(|(_, _, a), (_, _, b)| a.fill_ratio.partial_cmp(&b.fill_ratio) );
Burst injection
Watch adaptive re-routing
Click Inject burst to fire 20 tasks at once. Slow workers (4–7) saturate immediately — their fill bars turn red and the opacity governor emits BOUNDARY_OPAQUE. Within one peer_load cycle (~300 ms), dispatchers read the updated load map and start routing exclusively to the faster workers, draining the slow workers' queues passively.
// dispatcher: route to Individual scope let worker_id = worker.node_id().clone(); dispatcher.emit( "task", SignalScope::Individual(worker_id), task_payload, );
