Epidemic discovery
KV is the registry
Every register_mcp_tool call writes tools/{name}/{node_id} into the gossip KV store. Any node can discover any tool by scanning that prefix — no central registry, no configuration file. Add a node and its tools appear within one gossip round. Kill it and the tombstone propagates automatically.
// writes tools/weather/127.0.0.1:55003 let _h = agent.register_mcp_tool( "weather", schema, handler); // any node discovers it let entries = agent.scan_prefix( "tools/weather/"); // returns all provider node_ids
Load balancing
Multiple providers = automatic spread
Add a second node with the same tool. The caller scans tools/{name}/ and gets multiple entries — one per provider. It picks randomly among them. No coordinator needed. Kill one provider and calls drain to the survivor. Restart and load re-balances automatically — the KV mesh is the only state.
// KV scan returns all providers let entries = caller.scan_prefix( "tools/calculate/"); // 2 entries → 2 live providers // pick one uniformly at random let pick = now_ms() % entries.len(); let provider = entries[pick].parse()?;
Broker-less routing
Caller → provider direct
There is no router, no load-balancer, no service mesh proxy. The caller node reads the KV store locally (O(1)), emits an mcp.invoke signal directly to the provider via rpc_call, and awaits the reply. The arc on the canvas is the actual signal path — watch it change as different callers and providers are selected each cycle.
// direct point-to-point — no hop let reply = caller.rpc_call( provider, "mcp.invoke", request_bytes, Duration::from_secs(5), ).await?; // reply arrives from provider directly
