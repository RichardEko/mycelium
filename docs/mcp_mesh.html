Layer 4 — MCP Bridge
No registry. No broker.
Every tool is written under tools/{name}/{node_id} in the gossip KV store. Any node can discover any tool by scanning that prefix — the substrate is the registry. Kill a node and its tools vanish from the mesh in one gossip round; restart and they reappear. No operator action needed.
// Register a tool — writes KV entry let _handle = agent.register_mcp_tool( "weather", schema, |args| async move { /* handler */ }, ); // Drop handle → tombstone propagates
Epidemic discovery
Any caller, any provider
A tools/call request arrives at any node. That node scans the gossip KV for a provider, extracts its NodeId, and issues an RPC directly — a point-to-point mcp.invoke signal routed through the mesh. Watch the arcs: the caller and the provider are almost never the same node.
// Caller scans for any provider let entries = agent.scan_prefix("tools/weather/"); let provider = entries[0].0.parse::<NodeId>()?; // Point-to-point RPC through mesh let reply = agent.rpc_call( provider, "mcp.invoke", request_bytes, timeout, ).await?;
Self-healing
Kill it. Restart it. Done.
There is no configuration to update, no service to notify. When node-0 dies its McpToolHandle is dropped, which fires a cancel channel, which runs the tombstone path: a KV delete entry propagates to every peer. When node-0 restarts it re-registers — the same KV path, a newer HLC timestamp — and the mesh self-corrects.
// Kill: just drop the handle drop(tool_handle); // tombstone fires // Restart: new agent, new handle let agent = GossipAgent::new(id, cfg); agent.start().await?; let _h = agent.register_mcp_tool( "weather", schema, handler);
